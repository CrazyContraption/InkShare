InkShare

Welcome, and thank you for participating in using InkShare!

InkShare allows users to collaborate on drawing projects from anywhere via Socket.io and P5 integrations with Angular.

InkShare uses five fundamental but seperate parts divided over three servers to operate.

Below you can find the three fundamental parts and how they losely operate.
Fundamental Parts
  1. Website - Frontend user UI provided via Angular/cli encased in a Node.js wrapper.
  2. Webservers - Connects canvases together in real time via Socket.io
  3. Canvas App - Implemented via P5 and minor JavaScript black magic.
  4. Webservice - Security bridge to the database using get and put requests.
  5. Database - Backend data storage provided via a MongoDB service.
  Servers
  1. Website - Heroku Server 1
    • Contains the frontend that the user sees and operates with.
    • Uses Angular routing, forum validation to communicate with our webservice. This includes:
      • Creating accounts
      • Logging in
      • Canvas handoffs
    • Uses Socket.io and as js node wrapper to display a dynamic P5 canvas.
    • Features authentication and privacy components.
    • Dynamically generates additional Socket.io servers for every new canvas, hosted locally on the same server.
  2. Webservice - Heroku Server 2
    • Uses get and put requests from the Website to securely communicate with the database for the Website.
    • Handles the security of the database by not allowing clients to directly interact with the data.
  3. Database - MongoDB Server
    • Hosts user accounts
    • Hosts canvases
    • Communicates with the webservice via a backend schema.
